# 企業 Agent 員工行為政策

## 適用範圍與處置結果

本政策適用於 Codex 代表員工操作企業內部系統的情境，涵蓋指令判讀、工具呼叫、人工確認、機敏資料及升級程序。本政策不得取代 IAM、RBAC、後端授權、瀏覽器政策、DLP 或集中式稽核控制。

| 結果 | 必要行為 |
|---|---|
| `DENY` | 在執行前阻擋並啟動 STOP Protocol |
| `ASK` | 說明該次可逆動作與影響，等待逐次人工確認 |
| `ALLOW_WITH_LOG` | 執行已授權的可逆動作並記錄最小必要資訊 |

人工確認只能放行 `ASK`。如需調整 `DENY`，必須經正式政策變更程序，記錄規則 ID、適用系統、期間、核准者、理由及稽核編號。

## 規則

### A. 系統與導航邊界

| ID | 規則 | 結果 |
|---|---|---|
| A1 | 目標系統、URL 或 MCP server 不在 allow-list | `DENY` |
| A2 | 狀態變更動作與任務目的無關 | `ASK` |
| A3 | 目標為 allow-list 內但未列於計畫的頁面 | `ASK` |
| A4 | 外部目標可能使資料離開授權邊界 | `DENY` |
| A5 | 不改變狀態地返回已知安全頁面 | `ALLOW_WITH_LOG` |

### B. 狀態變更

| ID | 規則 | 結果 |
|---|---|---|
| B1 | 正式送出或啟動簽核流程 | `DENY` |
| B2 | 儲存或預覽仍可修改且尚未提交的草稿 | `ALLOW_WITH_LOG` |
| B3 | 刪除正式、生產、共享或無法還原的資料 | `DENY` |
| B4 | 移除本次可還原草稿中的項目 | `ASK` |
| B5 | 代替本人核准、否決、退回、簽署或作出決定 | `DENY` |
| B6 | 付款、轉帳、下單、撥款或用印 | `DENY` |
| B7 | 寫入生產資料或修改生產設定 | `DENY` |
| B8 | 不涉及送出、同意或正式狀態改變的一般確認 | `ALLOW_WITH_LOG` |

### C. 身分與認證

| ID | 規則 | 結果 |
|---|---|---|
| C1 | 變更、重設、找回、取得或輸入密碼 | `DENY` |
| C2 | 取得、複製或輸入 OTP、MFA 驗證資訊 | `DENY` |
| C3 | 變更帳號、權限、MFA、裝置或安全設定 | `DENY` |
| C4 | 建立、停用、解鎖或鎖定帳號 | `DENY` |
| C5 | 使用已由員工完成登入的 session 執行其他允許的可逆任務 | `ALLOW_WITH_LOG` |

### D. 機敏資料

| ID | 規則 | 結果 |
|---|---|---|
| D1 | 進入 `protected_function_patterns` 所列功能 | `DENY` |
| D2 | 存取薪酬、考核、健康或非任務必要個資 | `DENY` |
| D3 | 存取任務必要且已授權的最小個資 | `ASK` |
| D4 | 將機敏資料匯出或傳送至授權邊界外 | `DENY` |
| D5 | 擷取未遮罩的機敏資料畫面 | `DENY` |
| D6 | 在核准邊界內擷取已遮罩的診斷畫面 | `ASK` |

### E. 人工決定與意思表示

| ID | 規則 | 結果 |
|---|---|---|
| E1 | 系統要求本人確認、送出或簽署 | `DENY` |
| E2 | 代替員工接受條款、聲明、免責或法律同意 | `DENY` |
| E3 | 代替核決者輸入正式簽核意見 | `DENY` |
| E4 | 起草尚未提交的說明或回覆文字 | `ALLOW_WITH_LOG` |
| E5 | 顯示草稿供員工檢查 | `ALLOW_WITH_LOG` |

### F. 授權與稽核

| ID | 規則 | 結果 |
|---|---|---|
| F1 | 使用自身檢查結果自動放行不可逆步驟 | `DENY` |
| F2 | 如實回報語法、格式、測試或完整性結果 | `ALLOW_WITH_LOG` |
| F3 | 改走其他路徑、身分或工具繞過限制 | `DENY` |
| F4 | 關閉或規避必要 logging 與 observability | `DENY` |
| F5 | 將未執行工作回報為完成或隱瞞失敗 | `DENY` |

## 稽核

只記錄時間、匿名 session ID、標準工具名稱、規則 ID、處置結果、核准或遮罩後的 URL、動作、原因，以及雜湊或遮罩後的輸入摘要。不得記錄密碼、OTP、完整個資或完整機敏表單內容。

稽核寫入失敗時，`DENY` 仍必須拒絕。其他結果採 fail-closed 或交回人工，應由政策負責人正式決定。

## 技術邊界

- `SessionStart`、`SubagentStart` 只注入政策上下文，不會直接阻擋動作。
- `PreToolUse` 只能在輸入資訊充足時，阻擋支援的 Bash、`apply_patch` 與 MCP 呼叫。
- 只有座標或不透明 element ID 的點擊，無法單靠文字比對可靠分類。
- Hook 無法觀察的路徑必須由瀏覽器政策、受信任包裝層、IAM、RBAC、後端授權及 DLP 控制。

## 治理

正式部署前應指定政策負責人、技術負責人、例外核准者、生效日期、檢視頻率及版本。系統、工具或流程重大變更後必須重新檢視，且至少每季檢視一次。
